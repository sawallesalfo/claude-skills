# Branding Guide — arch-diagram

How to apply your company's color palette to any diagram generated by this skill.

---

## How It Works

Every diagram uses **3 color roles**:
1. **Primary** — main lane / dominant process flow
2. **Secondary** — supporting lane / alternate path
3. **Neutral** — output lane / storage / grayed-out sections
4. **Title bar** — header strip (usually your brand's darkest color)

Replace those 4 roles with your brand colors. Everything else (arrows, text, warning badges) stays neutral.

---

## Palette Template

```yaml
# brand-palette.yaml — fill in your colors
name: "Your Company Name"
primary:
  fill:   "#e8f5ee"   # light tint of brand color 1
  stroke: "#00843D"   # brand color 1 (full)
  text:   "#00562A"   # dark shade for readability

secondary:
  fill:   "#fdf6e3"   # light tint of brand color 2
  stroke: "#C8962A"   # brand color 2 (full)
  text:   "#7a5c16"

neutral:
  fill:   "#f8fafc"
  stroke: "#cbd5e1"
  text:   "#475569"

title_bar: "#00843D"  # usually brand color 1 (dark)
title_text: "#ffffff"
```

## Prompt Instruction

When asking Claude to generate a branded diagram, add to your prompt:

> Use the following brand palette:
> - Primary: fill `#e8f5ee`, stroke `#00843D`, text `#00562A`
> - Secondary: fill `#fdf6e3`, stroke `#C8962A`, text `#7a5c16`
> - Title bar: `#00843D` background, white text

Or simply name a preset:
> Use the **Green & Gold** palette from `references/branding.md`

## Deriving Tints Automatically

If you only have your brand's primary hex color, derive the palette:
- **Fill** (light tint): mix with white at 90% — use a tool like `tinycolor2` or CSS `color-mix(in srgb, #00843D 12%, white)`
- **Stroke** (full): use the brand color directly
- **Text** (dark shade): darken by ~30% for contrast — target WCAG AA (4.5:1 ratio on white)
